# Wayli Monetisation Ideas

## Core Principle

Free users should get genuine value.

Paid users should get greater clarity, deeper personalised insights, and stronger decision confidence.

Wayli should never feel like a crippled calculator behind a paywall.

---

# Monetisation Model V1

## Free Tier

Show 3 personalised insights based on user inputs.

Examples:

- Full repayment looks unlikely under current assumptions
- £100/month extra could shorten repayment by 4 years
- You may need ~£56k salary to clear within the loan term

Goal:

- Build trust
- Create shareable value
- Encourage repeat use
- Capture emails

---

## Paid Tier

## Full Analysis Unlock

Suggested price:

- £4.99
- £7.99
- £9.99 test ranges

One-time unlock for current scenario.

Includes:

- All personalised insights (10+)
- Salary rise scenarios
- Overpayment scenarios (£50 / £100 / £200 / custom)
- Interest outlook
- Total likely repayment scenarios
- What changes the result most
- Practical next-step suggestions
- Downloadable summary / PDF
- Save results

Goal:

Help users feel confident making a real decision.

---

# Subscription Model (Later)

## Wayli Premium

Suggested price:

- £4.99/month
- £29/year

Includes access to all tools:

- Student Loan Tool
- Mortgage Tool

# Email Funnel Option

## Free User Journey

3 free insights shown.

Prompt:

Enter email to unlock 2 extra insights.

Then offer premium full report.

Good for low-friction list growth.

---

# Product Positioning

Do not sell:

- More numbers
- More charts
- Complex reports

Sell:

- Confidence
- Clarity
- Faster decisions
- Peace of mind
- Personalised understanding

# Commercial Targets Example

10,000 monthly visitors

- 30% complete tool = 3,000 users
- 5% email signup = 150 leads
- 2% buy £7.99 unlock = 60 sales
- ~£480/month from one tool

With multiple tools + optimisation this scales.

---

# Important Rules

1. Free tier must feel useful.
2. Paid tier must feel meaningfully better.
3. Never use fear tactics.
4. Keep tone calm and practical.
5. Sell confidence, not complexity.
